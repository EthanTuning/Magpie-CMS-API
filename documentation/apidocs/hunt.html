get /hunts

Description

Retrives a list of Hunts that match the search criteria. This method does not return complete Hunts, only the first level data.

Example

When called without parameters, returns all available Hunts to the current user.

http://magpiehunt.com/api/v1/hunts

Any user-editable attribute can be a query parameter.

http://magpiehunt.com/api/v1/hunts?state=WA

Parameters can also be chained together.

http://magpiehunt.com/api/v1/hunts?state=WA&city=Seattle

Up 
post /hunts

Description

Adds a Hunt to the database.

Example

There is no minimum required data to make a Hunt. Blank hunts are allowed. Also, any fields not listed below are ignored in the request.

The following fields are allowed in Body: x-www-form-urlencoded

"abbreviation": string, "audience": string, "date_end": YYYY-MM-DD, "date_start": YYY-MM-DD, "name": string, "ordered": to be determined, "summary": string, "sponsor": string, "super_badge": string, "city": string, "state": 2-letter state code, "zipcode": string

Return type

Currently it just returns the hunt_id of the hunt created.

Content-Type: application/json

		{ "hunt_id": "668" }
]

put /hunts/{hunt_id}

Description

Update a Hunt

Example

Works the same as POST, but require a Hunt ID in the URL. Fields entered will overwrite existing fields, and any fields left blank will delete existing fields. This method just blasts out an existing Hunt with the data sent. It does NOT incrementally update.

http://magpiehunt.com/api/v1/hunts/668
