Magpie API Documentation

This page is a developer guide for the Magpie web Application Programming Interface (API). The API attempts to be as RESTful as possible, but some exceptions were made in order to benefit the client application(s). For example, performing a GET request to obtain a single Hunt will return all the child objects belonging to that Hunt as well as the Hunt itself.

Overview

The API is a portal for users to send and receive data to Magpie. The API does not distinguish between client types, web and mobile clients are given the same responses. Hunts are the primary resource. They contain their own datafields, as well as act as containers for the sub-resources (Badges and Awards). Sub-resources are added to a Hunt piecewise. However, Hunts are retreived in a complete form.

Access

All requests to the API must contain an Authorization Header with a Bearer token. The token is aquired from a Google login flow.

  • Authorization: Bearer "Token String"
